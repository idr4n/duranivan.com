# Architecture

Technical overview of the duranivan.com codebase.

## Overview

The site is an Astro 6 static site consisting of a single page. It produces plain HTML, CSS, and a minimal client-side script for the theme toggle — no frontend framework is loaded at runtime.

## Directory Structure

```
src/
├── layouts/
│   └── Layout.astro     # HTML shell, <head> meta, flash-prevention script
├── pages/
│   └── index.astro      # Page content, client-side theme logic, scoped styles
└── styles/
    └── global.css       # CSS reset, theme tokens, base styles

public/                  # Static assets served as-is (favicon, robots.txt, CNAME)
dist/                    # Build output (generated by `astro build`)
```

### `src/layouts/Layout.astro`

The HTML shell shared by all pages. Responsibilities:

- Imports self-hosted font files (`@fontsource/ibm-plex-mono` at 400, 500, 700)
- Imports `global.css`
- Sets `<meta>` tags (description, OpenGraph, viewport)
- Contains an **inline `<script>`** that runs before first paint to prevent theme flash

### `src/pages/index.astro`

The single page of the site. Contains:

- HTML markup for header, name, tagline, links, coming-soon section, and footer
- A client-side `<script>` block managing the theme toggle UI, preference persistence, and live OS preference tracking
- Scoped `<style>` block with all page-specific CSS, including responsive breakpoints

### `src/styles/global.css`

Global stylesheet providing:

- Universal box-sizing reset
- Theme token definitions via `html[data-theme='dark']` and `html[data-theme='light']` selectors
- CSS custom properties: `--bg-primary`, `--bg-secondary`, `--text-primary`, `--text-secondary`, `--border`, `--accent`
- Base typography (IBM Plex Mono) and body styles

## Theme System

The theme system uses a two-layer approach:

### Layer 1 — Flash Prevention (inline script in `<head>`)

An inline script executes synchronously before the browser paints. It:

1. Reads `localStorage.getItem('theme')`
2. If the value is `'dark'` or `'light'`, uses it directly
3. Otherwise (including `'system'`, `null`, or any invalid value), resolves the OS preference via `matchMedia`
4. Sets `document.documentElement.setAttribute('data-theme', resolvedTheme)`

This ensures the correct theme is applied before any content renders, avoiding a flash of the wrong theme.

### Layer 2 — Client-side Toggle (script in `index.astro`)

A standard `<script>` block manages interactive behavior:

- **`getStoredPreference()`** — reads and validates the stored theme value; normalizes invalid values to `'system'`
- **`applyTheme(value)`** — resolves `'system'` to the current OS preference, then sets `data-theme`
- **`updateToggleUI(activeValue)`** — highlights the active button in the light/dark/system toggle
- **Click handlers** — persist the chosen value to `localStorage` and apply it immediately
- **`matchMedia` change listener** — re-applies the theme when the OS preference changes and the stored preference is `'system'`

### Default Behavior

On first visit (no stored preference), the site follows the OS color scheme preference. The toggle shows "system" as active.

## Typography

The site uses **IBM Plex Mono**, self-hosted via `@fontsource/ibm-plex-mono`. Three weights are loaded:

- 400 (regular) — body text
- 500 (medium) — labels and header elements
- 700 (bold) — name heading

## Build Output

Running `astro build` produces static HTML, CSS, and JS files in the `dist/` directory. The output is fully self-contained with no server-side runtime required.

## Deployment

The GitHub Actions workflow at `.github/workflows/deploy.yml` handles deployment:

1. Triggers on push to the `main` branch
2. Installs dependencies and runs `astro build`
3. Deploys the `dist/` directory to GitHub Pages

The custom domain `duranivan.com` is configured via the `public/CNAME` file.
